The Mitigation Controller wraps everything. It short-circuits blocked IPs before wasting detection cycles, logs every flagged event to SQLite, manages timed blocks with auto-expiry, and escalates repeat offenders.

Escalation: deprioritise strikes → auto-block

A single anomalous request is deprioritised (forwarded but tagged). Repeated anomalies from the same IP indicate sustained suspicious activity. The escalation system tracks strikes in a sliding window — when the count reaches the threshold, the IP is automatically upgraded to a full timed block.

►
Strike threshold: 8 strikes
Why not 1? Why not 20?
Set to 8 to provide a small buffer against jitter false positives (occasional high-Z requests that are not attacks) while still firing reliably when 50 concurrent burst requests arrive. A threshold of 1 would block on any anomalous request, causing false positives during normal traffic variation.
►
Window: 120 seconds
Old strikes expire and don't haunt the IP
A brief network anomaly generating 3 strikes should not result in a block 10 minutes later. The 120s sliding window means only recent strikes count. Strikes older than 120s are evicted from the deque, giving the IP a clean slate over time.
►
Block duration: 30 seconds
Short for demo repeatability; production would be 5–15 min
30 seconds lets the demo be repeated without long waits. In a production system, escalation blocks would typically be 5–15 minutes. The duration is configurable via the /idms/config endpoint.
