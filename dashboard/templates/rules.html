Load Balancer
The Rule Engine runs first on every request. It checks rules in strict priority order and short-circuits on the first match — an unauthenticated request is blocked immediately without ever reaching SQL injection checks or rate limiting. If no rule fires, the request passes to the Anomaly Engine.
Detection rules (checked in priority order)
PRIORITY 1 API Key Validation
Every request must include a valid X-API-Key header. Missing, malformed (under 8 chars, contains injection characters), or incorrect keys are blocked instantly. This is the sole authentication boundary — unauthenticated traffic never reaches any other check.
Action: Block   Severity: medium
PRIORITY 2 SQL / NoSQL Injection
Recursively extracts all strings (keys and values) from the JSON body and matches against 10 compiled regex patterns: UNION SELECT, DROP TABLE, OR 1=1, MongoDB operators ($where, $ne), time-based blind (SLEEP), comment markers (-- # /*), and more.
Action: Honeypot   Severity: critical — attacker gets fake 200 response
PRIORITY 3 Rate Limiting
Sliding-window counter per IP. Timestamps are stored in a deque and evicted as they leave the window. When the count exceeds the threshold, the IP is blocked. The decision is made inside the lock to prevent race conditions at the threshold boundary.
Config: 200 requests / 10 seconds   Action: Block
PRIORITY 4 Endpoint Scan
Detects rapid probing of multiple distinct API paths from a single IP. Attackers enumerating the API surface hit many different endpoints quickly — legitimate clients typically use 1–2 paths.
Config: >5 distinct paths in 10s   Action: Deprioritise
PRIORITY 5 Payload Size
Two sub-checks: (a) raw body size exceeding 8 KB, and (b) sort_large_list requests with more than 500 items. Prevents memory exhaustion attacks where oversized payloads consume server resources.
Config: 8192 bytes / 500 items   Action: Block
PASS All Checks Passed
If none of the above rules fire, the request is clean from a rule perspective. It then moves to the Anomaly Engine (Phase 2) for statistical scoring against the IP's baseline behaviour.
Action: Forward to Anomaly Engine
